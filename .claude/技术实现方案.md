# 技术实现方案

## 项目信息

- **名称**：claude-weixin（Claude Code WeChat 桥接）
- **运行时**：Bun >= 1.0
- **语言**：TypeScript (ESNext modules)
- **依赖**：`@modelcontextprotocol/sdk`、`qrcode-terminal`、`zod`

## 目录结构

```

├── daemon.ts            ← 主入口：HTTP 服务 + iLink 长轮询 + Claude CLI 调度
├── mcp-adapter.ts       ← MCP 适配器：stdio→HTTP 桥接，供 Claude Code 调用
├── cli.ts               ← CLI 路由入口
├── setup.ts             ← 微信扫码登录 + 白名单管理
├── doctor.ts            ← 状态诊断（登录、白名单、MCP 配置）
├── types.ts             ← 接口和常量定义

├── ilink-api.ts         ← ilink HTTP API 封装（getupdates、sendmessage）
├── media.ts             ← AES 加解密、CDN 下载上传、媒体内容提取
├── chat-log.ts          ← 聊天记录持久化（JSONL）和回放
├── heartbeat.ts         ← 心跳调度（定点 + 随机）
├── package.json         ← 依赖和脚本

├── heartbeat-config.example.json
├── README.md

├── .claude/             ← 项目文档
├── 一键扫码登录.cmd
├── 一键状态检查.cmd
├── 一键安装MCP配置.cmd
└── 一键启动守护进程.cmd
```

## 架构

### 核心流程

```
微信 ClawBot
    ↓ (ilink HTTP 长轮询)
daemon.ts (守护进程)
    ├── HTTP Server (localhost:18923)
    │   ├── /api/reply          → 回复文字
    │   ├── /api/send-file      → 发送文件
    │   ├── /api/reload-heartbeat → 刷新心跳
    │   └── /api/ping           → 健康检查
    ├── iLink Polling Loop      → 拉取微信消息
    └── Claude Process Manager  → 调度 claude -p
            ↓ (stdio / MCP)
    mcp-adapter.ts (MCP Server)
            ↓ (HTTP → daemon)
    Claude Code (claude -p --mcp-config)
```

### 消息处理流程

1. daemon.ts 通过 ilink getupdates 长轮询拉取消息
2. 解析消息类型（文字/图片/文件/视频/语音），下载媒体文件
3. 白名单校验（未授权 sender 静默丢弃）
4. 构造 prompt（含聊天历史回放），调用 `claude -p`
5. Claude 通过 MCP 适配器调用 wechat_reply/wechat_send_file 工具
6. 工具调用 → mcp-adapter.ts → HTTP → daemon.ts → ilink sendmessage → 微信

### 关键设计

- **独立守护进程**：不依赖 Claude Code Channels 协议，通过 HTTP + MCP 自行桥接
- **聊天记忆**：重启时回放最近 200 条聊天记录（chat_history.jsonl）
- **心跳调度**：定点 + 随机时间表，修改配置自动热加载
- **白名单门控**：基于 sender_id 的访问控制，支持自动添加模式

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `WEIXIN_DAEMON_PORT` | 18923 | 守护进程 HTTP 端口 |
| `WEIXIN_DIR` | ~/.claude/weixin | 数据目录 |

## 数据文件（~/.claude/weixin/）

| 文件 | 内容 |
|------|------|
| account.json | 微信 Bot Token（登录后生成） |
| allowlist.json | 授权用户白名单 |
| sync_buf.txt | ilink 同步状态 |
| context_tokens.json | 各 sender 的 context_token |
| chat_history.jsonl | 聊天记录（JSONL） |
| heartbeat-config.json | 心跳配置 |
| media/ | 下载的媒体文件 |

## 代码规范

- 运行前不需要编译，Bun 直接执行 .ts 文件
- import 使用 `.js` 扩展名（Bun 模块解析约定）
- 日志统一输出到 stderr（不影响 stdout）
- 日志中有 token/key 的地方用 sanitizeText 脱敏
- 所有文件操作使用 `node:` 前缀的 Node.js API

## 已知技术陷阱

- ilink API 无官方 SLA，接口可能随时变更
- context_token 会过期，需定期重新获取
- `claude -p` 的 `--resume` flag 和 MCP channel flag 不兼容
- Windows 下 `fs.watch` 行为可能与 macOS/Linux 不同（心跳热加载用）
- Bun 的 crypto 模块与 Node.js 行为一致，可放心使用
